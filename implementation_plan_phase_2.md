# Implementation Plan: Phase 2 (Social & Cloud)

## 1. Архитектурный рефакторинг (Подготовка к масштабированию)

### 1.1 Переход на Динамические Цели (N-Goals)
**Цель:** Уйти от жесткой привязки `goal_a` / `goal_b`.
*   **БД:** Удалить `goalAAmount`, `goalBAmount` из таблицы `Deposits`.
*   **БД:** Создать таблицу `DepositAllocations`:
    - `id` (UUID)
    - `depositId` (FK -> Deposits.id)
    - `goalId` (FK -> Goals.id)
    - `amount` (Int, minor units)
*   **Код:** Обновить `SavingsNotifier.createDeposit` для работы со списком аллокаций.
*   **UI:** Обновить `DepositScreen` для поддержки выбора N целей (чекбоксы/слайдеры).

### 1.2 Оптимизация для Синхронизации (Sync Readiness)
*   **БД:** Добавить колонку `updatedAt` (DateTime) во все таблицы для стратегии "Last Write Wins".
*   **БД:** Убедиться, что ВСЕ таблицы используют UUID в качестве первичного ключа (кроме системных настроек).
*   **Service:** Создать `SyncService` (интерфейс), который будет отвечать за отправку локальных изменений в облако.

## 2. Облачная синхронизація (Firebase + CRDT Lite)

### 2.1 Интеграция Firebase
*   Использовать **Cloud Firestore** для хранения данных Squads и совместных целей.
*   Использовать **Firebase Auth** для идентификации пользователей.

### 2.2 Стратегия разрешения конфликтов (Conflict Resolution)
Учитывая опыт с CRDT, предлагается гибридный подход:
*   **Personal Data:** Last Write Wins на основе `updatedAt`.
*   **Shared Goals/Squads (Counter CRDT):** Для сумм в совместных целях использовать инкрементальные обновления (+= delta), а не перезапись итоговой суммы, чтобы избежать потери данных при одновременных депозитах.

## 3. Соціальні функції (Squads 2.0)

### 3.1 Реализация Squads
*   **Real-time Feed:** Лента активности участников (кто сколько сохранил).
